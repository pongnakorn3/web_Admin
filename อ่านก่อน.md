ก๊อปปี้เนื้อหาด้านล่างนี้ไปใส่ในไฟล์ README.md ได้เลยครับ ผมสรุปขั้นตอนที่ถูกต้องเพื่อให้คุณและเพื่อนอ่านแล้วเข้าใจตรงกันและทำงานร่วมกันได้ทันทีครับ:

🚀 คู่มือการทำงานร่วมกันในโปรเจกต์ web_Admin
เพื่อให้โค้ดไม่ชนกันและทำงานได้อย่างเป็นระบบ ให้ทุกคนทำตามขั้นตอนดังนี้:

1. การเตรียมตัว (ทำครั้งแรกครั้งเดียว)
เจ้าของโปรเจกต์: ต้องไปที่ Settings > Collaborators เพื่อเพิ่มชื่อเพื่อน และเพื่อนต้องกดตอบรับในอีเมลก่อน

เพื่อน (Collaborator): ให้ Clone งานลงเครื่องด้วยคำสั่ง:

Bash
git clone https://github.com/pongnakorn1/web_Admin.git

2. การสร้างกิ่งใหม่เพื่อทำงาน (Branching)
สำคัญมาก: ห้ามแก้โค้ดในกิ่ง main โดยตรง ให้สร้างกิ่งใหม่ตามหน้าที่ที่ได้รับมอบหมายเสมอ:

สร้างและย้ายไปกิ่งใหม่:

Bash
git checkout -b ชื่อกิ่งของคุณ (เช่น feat-login หรือ fix-ui)
3. การบันทึกและส่งงาน (Commit & Push)
เมื่อแก้ไขโค้ดในเครื่องตัวเองเสร็จแล้ว ให้ส่งงานขึ้นไปดังนี้:

เลือกไฟล์ทั้งหมด: git add .

บันทึกสิ่งที่ทำ: git commit -m "อธิบายสั้นๆ ว่าแก้อะไรไป"

ส่งกิ่งของคุณขึ้น GitHub: git push origin ชื่อกิ่งของคุณ

4. การรวมโค้ดเข้ากิ่งหลัก (Merge)
เมื่อส่งกิ่งขึ้นไปแล้ว ให้ไปที่หน้าเว็บ GitHub แล้วกดปุ่ม "Compare & pull request" เพื่อให้เจ้าของโปรเจกต์ตรวจสอบและกด Merge รวมโค้ดเข้าสู่กิ่ง main

5. การอัปเดตโค้ดล่าสุด (Sync)
ก่อนเริ่มงานใหม่ในวันถัดไป ให้ดึงโค้ดล่าสุดที่เพื่อนคนอื่นทำไว้มาไว้ที่เครื่องตัวเองเสมอ:

Bash
git checkout main
git pull origin main
